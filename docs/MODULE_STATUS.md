# Module Status

This document tracks the current state of the major Olympus Command modules.

## Flight

### Current

- Aircraft map rendering.
- Emergency squawk/status handling.
- Manual air emergency reporting.
- Confirm Emergency action.
- Make Radio Contact action.
- Stand Down / Resolve action.
- Flight notifications.
- Global notification integration.
- Aviation infrastructure overlays.
- Airport and infrastructure popups.
- Atmospheric weather visualization.
- Diagnostic status bar.

### Desired next work

- Ensure Flight detail panel uses Maritime visual style.
- Ensure Flight detail panel is draggable/dockable.
- Ensure detail window updates on map item selection and closes on deselection.
- Add aircraft record/folder style comparable to Maritime vessel folders.

## Maritime

### Current

- AISStream integration.
- AIS diagnostics endpoint at `/api/maritime-local-status`.
- Vessel state cache.
- Vessel map display.
- Vessel detail drawer.
- Vessel folder display.
- Mayday/distress context panel.
- Static source-kind fields for vessel/base-station/navigation-aid concepts.

### Desired next work

- Add ship-shaped markers.
- Add static AIS/base-station markers.
- Add Sault Ste. Marie / Soo Locks static location.
- Add moored-count display wiring in the active Maritime page.
- Make Maritime detail panel draggable/dockable if not active in current build.
- Convert vessel data folders into real browsable records.

## CAD

### Current

- Local CAD service runs as `olympus-cad.service`.
- Health endpoint: `http://127.0.0.1:5050/health`.
- Persistent file: `/var/lib/dennco/olympus-cad/cad-state.json`.
- CAD state is persistent.
- CAD modules include personnel, calls, units, mapping, shifts, logs, reports, documents, calendar, notes, trainings, inventory, and dispatch queue.

### Desired next work

- Expand each CAD module with functional forms/actions.
- Add call folders and attachments.
- Add unit assignment workflow.
- Add map-linked CAD markers.
- Add Admin CAD settings integration.
