# Olympus Cyber Notes

## Planning status

Draft. This app pack exists to collect cyber-specific direction without pushing cyber-only assumptions into the Olympus core.

## Research questions

1. Should the first MVP focus on vulnerability alerts, asset risk, vendor advisories, or executive reporting?
2. Which official public cyber sources should be enabled first?
3. How should confidence, severity, and exploitability be displayed?
4. What source attribution is required?
5. What compliance exports should be planned?

## Candidate source lanes

- NVD
- CISA Known Exploited Vulnerabilities
- CISA advisories
- Vendor security advisories
- Public CVE feeds
- Organization-owned asset inventory
- Authorized ticketing/security data

## Developer reminder

Cyber-specific behavior should start inside this app pack or a cyber extension. Do not add offensive capabilities. Keep secrets and API tokens server-side only.
